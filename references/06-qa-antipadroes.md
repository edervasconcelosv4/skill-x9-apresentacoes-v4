# QA e Antipadrões

## Checklist de Qualidade

| Item | Critério |
|---|---|
| Narrativa | Cada slide tem uma mensagem principal |
| Ritmo | Há alternância entre denso, leve, seção e fechamento |
| Layout | Todos os slides usam layouts catalogados |
| Densidade | Não há vazio excessivo nem texto espremido |
| Legibilidade | Corpo principal em tamanho confortável |
| Contraste | Texto claro sobre fundo escuro |
| Alinhamento | Cards e títulos seguem grid |
| Editabilidade | Conteúdo relevante é elemento nativo no Google Slides |
| Consistência | Paleta, raio, bordas, labels e títulos seguem padrão |
| Fonte | Preferir Montserrat ou fallback coerente |

## Antipadrões

| Antipadrão | Por que reprova |
|---|---|
| Slide inteiro como imagem | Não é editável |
| Bullet list longa | Parece documento colado no slide |
| Espaço morto | Dá sensação de slide incompleto |
| Tabela nativa sem tratamento | Fica com cara escolar e pouco premium |
| Texto pequeno demais | Não funciona em reunião |
| Cards desalinhados | Passa sensação de improviso |
| Fundo claro genérico | Perde estética Genspark/V4 |
| Ícones aleatórios | Polui e enfraquece a hierarquia |
| Cores demais | Quebra padrão executivo |
| Elementos sobrepostos | Torna a apresentação frágil e amadora |

## Teste de Entrega

Antes de considerar pronto:

1. Abra o HTML e percorra todos os slides.
2. Confira se títulos e cards não vazam.
3. Confirme se o IR tem os mesmos slides esperados.
4. Na versão Google Slides, tente editar um título, um card, uma métrica e um fluxo.
5. Se algum slide depender de screenshot para conteúdo central, refaça como nativo.
