Camada 1 — Contrato

IR JSON (deck.ir.json)

A representação intermediária. Define cliente, acento, tema e todos os slides com layout e conteúdo tipado. É a fonte de verdade — HTML e PPTX nascem dele.

Camada 2 — Preview

render-html-from-ir.js

Lê o IR e gera o HTML navegável. Tem limpeza automática de encoding PT-BR, templates por layout e suporte a drawflow posicionado com CSS custom properties.

Camada 3 — Entrega

render-pptx-from-ir.js

Usa pptxgenjs para criar PPTX com shapes nativos, fontes, cores de marca e tints calculados. Abre direto no Google Slides sem perder estrutura.

Setup

Como Instalar em Cada Modelo

A skill é um sistema de arquivos + instrução de comportamento. Cada plataforma tem um jeito de carregar isso.

⭐
Melhor opção. Claude Code acessa arquivos locais nativamente, executa Node.js e tem o SKILL.md como instrução de comportamento via projeto. É o ambiente em que a skill foi desenvolvida.

Passo a passo

  1. Abra Claude Code no diretório onde você instalou a pasta Skill X9/
  2. O arquivo SKILL.md dentro de Skill X9/ já é lido automaticamente quando você menciona apresentações ou decks
  3. Instale Node.js se ainda não tiver: nodejs.org
  4. Para PPTX, certifique-se que um projeto com pptxgenjs está disponível: 
    cd "pasta-do-seu-projeto"
npm install pptxgenjs
  5. Peça o deck: Claude vai gerar o IR, rodar os renderers e entregar HTML + PPTX

Comandos úteis

PowerShell
# Gerar HTML a partir de um IR
node "Skill X9\scripts\render-html-from-ir.js" `
  "meu-deck.ir.json" `
  "meu-deck.preview.html"

# Gerar PPTX a partir de um IR
node "Skill X9\scripts\render-pptx-from-ir.js" `
  "meu-deck.ir.json" `
  "meu-deck.pptx"

Node.js — Claude API
const Anthropic = require("@anthropic-ai/sdk");
const fs = require("fs");

const client = new Anthropic();
const skillMd = fs.readFileSync("Skill X9/SKILL.md", "utf8");
const layoutsMd = fs.readFileSync("Skill X9/references/03-layouts.md", "utf8");

const response = await client.messages.create({
  model: "claude-opus-4-7",
  max_tokens: 8192,
  system: skillMd + "\n\n" + layoutsMd,
  messages: [{
    role: "user",
    content: "Crie um deck de 4ª semana para [CLIENTE]. Retorne apenas o deck.ir.json."
  }]
});

const ir = JSON.parse(response.content[0].text);
fs.writeFileSync("deck.ir.json", JSON.stringify(ir, null, 2));
// Depois rode: node "Skill X9/scripts/render-html-from-ir.js" deck.ir.json preview.html

O accent é a única cor que muda por cliente. Vermelho é o padrão V4. Sempre especifique o HEX exato no IR: "accent": "#CC0000"
Regra 60/30/10
60% — fundo escuro (#050505, #0d0d0d, #131313)
30% — texto em cinza (#fff, #e0e0e0, #ccc, #9ca3af)
10% — acento do cliente (bordas, eyebrows, ícones, tags)
Tint de card obrigatório: nunca usar #1e1e1e puro como fundo. O renderer calcula cardTint = mixHex(CARD, accent, 0.12) — card cinza puro sem acento = slide morto.

Boas Práticas & Antipadrões

O que separa um deck executivo de um slide feito por IA sem critério.

✓ Faça sempre

✓
Gere o IR primeiro. Nunca saia direto para HTML sem o contrato estruturado.
✓
Use acentos corretos em PT-BR. "MÍDIA", "DISTRIBUIÇÃO", "intenção" — nunca sem acento.
✓
Todo slide tem uma função narrativa. Abrir, contextualizar, provar, comparar, decidir ou fechar.
✓
Paridade HTML → PPTX. Se o HTML tem drawflow, o PPTX precisa ter drawflow nativo — sem fallback de título vazio.
✓
Rode QA antes de entregar. Legibilidade, alinhamento, contraste, overflow e editabilidade.

✗ Antipadrões

🖼️
Slide inteiro como imagem
Não é editável. Reprova direto no critério de entrega.
📋
Bullet list longa como corpo
Parece documento colado. Transforme em cards, colunas ou etapas.
⬜
Espaço morto excessivo
Parece slide incompleto. Aumente hierarquia ou reorganize o grid.
🔤
Texto pequeno demais
Não funciona em reunião. Mínimo 13px para corpo de card.
⬜
Fundo branco genérico
Perde a estética dark premium. Sempre fundo escuro (#050505 a #151515).
📦
Drawflow em flex layout
Sem direção espacial. Usar position:absolute + CSS custom properties.
🎨
Cores fora da paleta
Quebra o padrão executivo. Só acento, success, warning e info além da paleta base.

Densidade Ideal por Slide

Slide Leve

1 ideia central

+ 2 a 3 apoios. Bom para teses, capas e big statements.

Slide Médio

3 a 5 cards

Diagnósticos, pilares e estratégias. O mais comum.

Slide Denso

6 a 9 blocos

Aceitável com grid claro. Máx. 90 palavras visíveis.

Slide Técnico

Fluxo ou matriz

Labels curtos, sem limite de blocos se houver estrutura visual.
